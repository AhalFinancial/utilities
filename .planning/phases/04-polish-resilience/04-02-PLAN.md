  <action>
Create `transcribe/checkpoint.py` with:

1. `TranscriptionCheckpoint` dataclass:
   - `video_path: str` — absolute path to source video
   - `video_hash: str` — MD5 hash of first+last 64KB (quick change detection)
   - `total_chunks: int` — expected number of chunks
   - `completed_chunks: List[int]` — chunk IDs that finished
   - `chunk_transcripts: Dict[int, str]` — chunk_id -> transcript text for completed chunks
   - `language: Optional[str]` — detected language (reuse on resume)
   - `model_size: str` — Whisper model used
   - `timestamp: float` — time.time() when saved

2. `calculate_file_hash(video_path: Path) -> str`:
   - Read first 64KB + last 64KB of file.
   - Return MD5 hex digest.
   - Handle files smaller than 128KB (just hash the whole file).

3. `get_checkpoint_path(video_path: Path) -> Path`:
   - Returns `video_path.parent / f"{video_path.stem}.checkpoint.json"`.

4. `save_checkpoint(checkpoint: TranscriptionCheckpoint, checkpoint_path: Path)`:
   - Write to `checkpoint_path.with_suffix('.tmp')` first using `json.dump(asdict(checkpoint), ...)`.
   - Then `os.replace(tmp, checkpoint_path)` for atomic write.
   - Clean up tmp on error.

5. `load_checkpoint(checkpoint_path: Path) -> Optional[TranscriptionCheckpoint]`:
   - Return None if file doesn't exist.
   - Try to load JSON and construct dataclass. Return None on JSONDecodeError/TypeError/KeyError.

6. `can_resume_from_checkpoint(checkpoint: TranscriptionCheckpoint, video_path: Path) -> bool`:
   - Verify `video_hash` matches current file hash.
   - Verify `str(video_path.resolve())` matches `checkpoint.video_path`.
   - Return True only if both match.

7. `delete_checkpoint(checkpoint_path: Path)`:
   - Delete if exists, silently ignore if not.

Update `transcribe/parallel.py`:

1. Import `save_checkpoint`, `TranscriptionCheckpoint`, `get_checkpoint_path`, `calculate_file_hash` from checkpoint.py.

2. Add optional parameters to `transcribe_chunks_parallel()`:
   - `checkpoint_path: Optional[Path] = None` — where to save checkpoints
   - `video_path: Optional[Path] = None` — original video path (for hash)
   - `completed_chunks: Optional[Dict[int, str]] = None` — already-done chunks from resume

3. In the executor loop, after each chunk completes successfully:
   - If `checkpoint_path` is provided, update and save a checkpoint with current state (completed chunk IDs + transcripts).

4. If `completed_chunks` is provided, skip submitting chunks whose IDs are in completed_chunks. Instead, create MergedSegment placeholders from the saved transcript text (these won't have perfect timestamps but the text will be correct).

5. For resumed chunks, the simplest approach: skip already-completed chunk IDs in the executor.submit loop, and after all new chunks complete, merge the saved transcript text with newly transcribed segments in chunk_id order.

IMPORTANT: Keep the existing function signature backwards-compatible — new parameters are all Optional with None defaults. Existing callers (cli.py) don't need changes until Task 2.

Update `transcribe/cli.py` with comprehensive error handling and resume support:

1. **Add --no-resume flag:**
   - Add `@click.option('--no-resume', is_flag=True, help='Start fresh even if checkpoint exists')` to the main command.

2. **Import new modules:**
   - `from transcribe.errors import TranscriptionError, APIKeyMissingError`
   - `from transcribe.checkpoint import (get_checkpoint_path, load_checkpoint, can_resume_from_checkpoint, delete_checkpoint, calculate_file_hash, TranscriptionCheckpoint)`

3. **Add resume detection after validation, before extraction:**
   - Call `checkpoint_path = get_checkpoint_path(video_path)`.
   - If `not no_resume`, try `load_checkpoint(checkpoint_path)`.
   - If checkpoint exists and `can_resume_from_checkpoint(checkpoint, video_path)`:
     - Display: "Found checkpoint: {len(checkpoint.completed_chunks)}/{checkpoint.total_chunks} chunks completed."
     - If not quiet, ask: `click.confirm("Resume from checkpoint?", default=True)`.
     - In quiet mode, auto-resume.
     - If user says no, `delete_checkpoint(checkpoint_path)` and continue fresh.
   - If checkpoint exists but is invalid (hash mismatch), display warning "Checkpoint found but video file has changed. Starting fresh." and delete it.

4. **Pass checkpoint info to parallel transcription:**
   - When calling `transcribe_chunks_parallel()` for multi-chunk paths, pass `checkpoint_path=checkpoint_path`, `video_path=video_path`, and `completed_chunks=checkpoint.chunk_transcripts` if resuming.

5. **Replace generic exception handling with custom error display:**
   - Replace the outer `except RuntimeError/ValueError/FileNotFoundError/Exception` blocks with:
     ```
     except TranscriptionError as e:
         e.display()
         sys.exit(1)
     except Exception as e:
         click.echo(click.style("Error: ", fg='red', bold=True) + str(e), err=True)
         click.echo(click.style("Suggestion: ", fg='yellow') + "Check that FFmpeg is installed and the video file is valid.", err=True)
         sys.exit(1)
     ```
   - This catches ALL custom errors via the base class and displays them with formatting.

6. **Delete checkpoint on successful completion:**
   - After writing the output file successfully, call `delete_checkpoint(checkpoint_path)`.

7. **Ensure errors always go to stderr:**
   - All `click.echo` calls for errors MUST use `err=True`.
   - Warnings always display regardless of `--quiet`.

8. **Update the validation calls** to NOT catch specific exceptions inline (since they now raise custom exceptions that the outer handler catches):
   - Remove the inner `try/except` around `validate_environment()` and `validate_video_file()` — let them raise their custom exceptions which the outer handler catches.
   - Remove the inner `try/except` around `check_api_key()` — same reason.

IMPORTANT: Keep the existing functionality intact. The --no-summary, --style, --force, --quiet flags must continue to work exactly as before. Only ADD resume capability and IMPROVE error display.

<task type="auto">
  <name>Task 2: Create audio chunking module with silence detection and deduplication</name>
  <files>transcribe/chunker.py</files>
  <action>
Create `transcribe/chunker.py` with the following components:

**1. `chunk_audio(audio_path, min_duration_for_chunking=600)` function:**
- Takes a Path to a WAV file and a minimum duration threshold (default 600s = 10 minutes)
- If audio duration < min_duration_for_chunking: return list with single chunk metadata `[(0, audio_path, 0.0, duration)]` — tuple of (chunk_id, path, start_time_offset, duration)
- If audio duration >= threshold:
  - Load audio with `pydub.AudioSegment.from_wav()`
  - Split on silence using `pydub.silence.split_on_silence(audio, min_silence_len=500, silence_thresh=-40, keep_silence=100)`
  - Merge small chunks to target ~20-25 seconds (iterate chunks, accumulate until >= 20000ms, then start new accumulation)
  - Add 2-3 second overlap: when creating final chunks, include last 3000ms of previous chunk at start of next chunk
  - Track start_time_offset for each chunk (cumulative duration minus overlap)
  - Export each chunk to a temp WAV file (use tempfile.mkdtemp() for a chunk directory)
  - Return list of `(chunk_id, chunk_path, start_time_offset, duration)` tuples

**2. `deduplicate_overlap(text_prev, text_next, overlap_words=50, threshold=0.85)` function:**
- Compare last `overlap_words` words of text_prev with first `overlap_words` words of text_next
- Use `difflib.SequenceMatcher` to find longest common match
- If match ratio > threshold, remove the overlapping portion from start of text_next
- Return cleaned text_next

**3. `cleanup_chunks(chunk_dir)` function:**
- Remove temporary chunk directory and all files within it
- Use shutil.rmtree for cleanup

**Important implementation details:**
- Use module-level functions (not class methods or lambdas) — required for pickle compatibility with ProcessPoolExecutor in Plan 03
- Log chunk count and sizes for debugging: use standard logging module
- Handle edge case: audio with no silence points (treat as single chunk)
- Handle edge case: very short chunks after splitting (merge with adjacent)

<task type="auto">
  <name>Task 2: Create Claude API summarizer with map-reduce, quality gate, and cost tracking</name>
  <files>transcribe/summarizer.py, pyproject.toml</files>
  <action>
**pyproject.toml:** Add `"anthropic>=0.84.0"` to dependencies list.

**transcribe/summarizer.py:** Create with these functions:

1. **`check_api_key() -> str`** — Check ANTHROPIC_API_KEY env var.
   - If not set: raise RuntimeError with message: "ANTHROPIC_API_KEY environment variable not set.\nGet your API key from: https://console.anthropic.com/settings/keys\nSet it with: export ANTHROPIC_API_KEY='your-key-here'"
   - Return the key string

2. **`calculate_cost(input_tokens, output_tokens, cache_read_tokens=0, cache_write_tokens=0) -> float`** — Claude Sonnet 4.6 pricing.
   - Base input: $3/M tokens
   - Output: $15/M tokens
   - Cache write: $3.75/M tokens
   - Cache read: $0.30/M tokens

3. **`split_into_semantic_chunks(text: str, max_tokens: int = 50000) -> list[str]`** — Split text at paragraph boundaries.
   - Split on '\n\n'
   - Estimate tokens: len(para) // 4
   - Accumulate paragraphs until approaching max_tokens
   - Always return at least one chunk

4. **`summarize_transcript(transcript_text: str, style: str, language: str = "en") -> tuple[str, int, int, float]`** — Core summarization.
   - Create `anthropic.Anthropic()` client (reads API key from env automatically)
   - Use model "claude-sonnet-4-6-20250514"
   - max_tokens=2048 for output
   - Use `get_system_prompt(style, language)` for system prompt
   - User message: "Please summarize this transcript:\n\n{transcript_text}"
   - Extract usage: input_tokens, output_tokens from response.usage
   - Calculate cost with calculate_cost()
   - Return (summary_text, input_tokens, output_tokens, cost_usd)

5. **`summarize_long_transcript(transcript_text: str, style: str, language: str = "en") -> tuple[str, int, int, float]`** — Map-reduce for long transcripts.
   - Use client.messages.count_tokens() to check token count
   - If under 150K tokens: delegate to summarize_transcript()
   - Otherwise: split_into_semantic_chunks(text, max_tokens=50000)
   - Map phase: summarize each chunk with max_tokens=1024, system prompt "Summarize this section of a longer transcript. Focus on key points, decisions, and topics."
   - Reduce phase: merge chunk summaries with get_system_prompt(style, language) and message "These are summaries of sections from a longer transcript. Create a cohesive {style} summary:\n\n{merged}"
   - Track total input/output tokens across all calls
   - Return (final_summary, total_input, total_output, total_cost)

6. **`summarize_with_quality_gate(transcript_text, confidence_pct, style, language, quiet=False) -> tuple[str, float, bool]`** — Quality gate wrapper.
   - Per user decision: threshold is 40%
   - If confidence_pct < 40 and quiet=False: use click.confirm("Low transcript confidence ({pct:.0f}%). Attempt summary anyway?", default=False)
     - If user declines: return ("", 0.0, False)
     - If user confirms: summarize and prepend disclaimer
   - If confidence_pct < 40 and quiet=True: click.echo warning, summarize anyway, prepend disclaimer
   - Disclaimer format: "> Note: Generated from low-confidence transcript ({pct:.0f}%). Verify key points.\n\n"
   - If confidence >= 40: summarize normally, return (summary, cost, True)
   - Internally call summarize_long_transcript() (handles both short and long transcripts)

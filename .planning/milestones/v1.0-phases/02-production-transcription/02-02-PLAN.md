<task type="auto">
  <name>Task 1: Upgrade transcriber with language detection, VAD, and quality validation</name>
  <files>transcribe/transcriber.py</files>
  <action>
Update the existing `Transcriber` class in transcriber.py with the following changes:

**1. Update `transcribe()` method signature:**
- Add optional `language` parameter (default None = auto-detect)
- Add `vad_filter=True` parameter to enable VAD filtering by default
- Add `vad_parameters=dict(min_silence_duration_ms=500)` to transcribe call

**2. Add `detect_language(audio_path)` method:**
- Transcribe first 30 seconds to detect language
- Return `(language_code, language_probability)` tuple
- If probability < 0.5, return `(None, probability)` to signal uncertain detection

**3. Add `validate_quality(segments)` method:**
- Calculate average of `segment.avg_logprob` across all segments
- Convert avg_logprob to a 0-100% confidence score for display: `confidence_pct = max(0, min(100, (1 + avg_logprob) * 100))`
  - This maps -1.0 → 0%, 0.0 → 100%, -0.5 → 50%
- Return `(is_acceptable, confidence_pct, avg_logprob)` where is_acceptable = avg_logprob >= -1.0

**4. Add `transcribe_with_quality(audio_path, language=None)` method:**
- Call `transcribe()` with language and VAD params
- Call `validate_quality()` on results
- If not acceptable AND self.model_size == "small":
  - Log warning: "Low confidence ({confidence_pct:.0f}%), retrying with medium model..."
  - Create new model with "medium" size, re-transcribe
  - Validate again
- Return `(segments, info, confidence_pct)` — always returns results regardless of quality

**5. Mark non-speech segments:**
- After transcription, iterate segments
- If `segment.avg_logprob < -1.5`, replace segment text with `[background noise]`
- If segment has `no_speech_prob > 0.9` (if available), also mark as `[background noise]`

**Important:** Keep the existing `transcribe()` method working as before for backward compatibility. The new `transcribe_with_quality()` is the upgraded entry point.
  </action>
  <verify>Run `python -c "from transcribe.transcriber import Transcriber; t = Transcriber(); print(dir(t))"` and confirm detect_language, validate_quality, transcribe_with_quality are present</verify>
  <done>Transcriber has detect_language, validate_quality, and transcribe_with_quality methods; VAD enabled by default; non-speech segments marked as [background noise]; auto-upgrade from small to medium on low confidence</done>
</task>

<task type="auto">
  <name>Task 2: Upgrade formatter with adaptive timestamps, paragraph merging, and metadata</name>
  <files>transcribe/formatter.py</files>
  <action>
Update formatter.py with the following changes:

**1. Replace `format_timestamp()` with `format_timestamp_adaptive(seconds, total_duration)`:**
- If total_duration >= 3600 (1 hour): return `H:MM:SS` format (no leading zero on hours)
- If total_duration < 3600: return `M:SS` format (no leading zero on minutes)
- Keep the old `format_timestamp()` function for backward compatibility but mark with comment `# Legacy: Phase 1 format`

**2. Add `merge_segments_to_paragraphs(segments, gap_threshold=2.0)` function:**
- Iterate segments, accumulate text into a paragraph
- When gap between current segment.end and next segment.start > gap_threshold seconds, emit paragraph
- Each paragraph is a tuple: `(start_timestamp, merged_text)`
- Strip and join segment texts with spaces
- This produces fewer, more meaningful timestamp markers at natural pauses

**3. Update `format_transcript()` to accept new parameters:**
- Add `confidence_pct` parameter (optional, default None)
- Add `model_name` parameter (default "small") to show which model was actually used (may be "medium" after auto-upgrade)
- Update metadata header to include:
  - `**Word count:** {N}` — count words in all segment text using `len(re.findall(r'\w+', text))`
  - `**Reading time:** ~{N} min` — word_count / 180 (rounded up)
  - `**Confidence:** {confidence_pct:.0f}%` — only if confidence_pct is not None
  - `**Model:** faster-whisper ({model_name})` — dynamic model name
- Use `merge_segments_to_paragraphs()` to group segments
- Format each paragraph block as:
  ```
  [MM:SS]
  Merged paragraph text here flowing naturally...

  ```
  (timestamp on its own line, paragraph text below, blank line between paragraphs)
- Use adaptive timestamp format based on total duration

**4. Add `import re` at top for word counting.**

**Important:** The function signature change must remain backward-compatible. New parameters have defaults so existing callers (cli.py) still work until Plan 03 updates them.

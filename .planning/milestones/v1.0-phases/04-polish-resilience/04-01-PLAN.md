Create `transcribe/errors.py` with a custom exception hierarchy:

1. `TranscriptionError(Exception)` — base class with `message: str`, `suggestion: Optional[str]`, and a `display()` method that outputs red "Error: " + message to stderr, plus yellow "Suggestion: " + suggestion if provided (using `click.style` and `click.echo(..., err=True)`).

2. Subclasses (each sets message + suggestion in __init__):
   - `FileValidationError(TranscriptionError)` — for corrupt/invalid files
   - `NoAudioStreamError(TranscriptionError)` — when video has no audio track (suggestion: "Check that the file has an audio track, or try a different file")
   - `CorruptedFileError(TranscriptionError)` — for corrupted files (suggestion: re-download or convert with ffmpeg)
   - `UnsupportedFormatError(TranscriptionError)` — for wrong format (suggestion: list supported formats)
   - `APIKeyMissingError(TranscriptionError)` — for missing OPENAI_API_KEY (suggestion: URL + export command)
   - `APIRetryExhaustedError(TranscriptionError)` — after all retries fail (accepts `attempts: int`, `last_error: str`; suggestion: wait and retry or check usage dashboard)
   - `FFmpegNotFoundError(TranscriptionError)` — for missing FFmpeg (suggestion: install commands per OS)

Update `transcribe/validators.py`:

1. Add import for `ffmpeg` and the new error classes from `transcribe.errors`.

2. Modify `validate_environment()`: Raise `FFmpegNotFoundError` instead of `RuntimeError`.

3. Modify `validate_video_file()`: After the existing extension check, add FFmpeg probe validation:
   - Call `ffmpeg.probe(str(video_path))` to validate file integrity.
   - Check that at least one stream has `codec_type == 'audio'`. If none, raise `NoAudioStreamError(video_path.name)`.
   - Catch `ffmpeg.Error` and parse stderr: if "Invalid data" or "End of file" in stderr, raise `CorruptedFileError(video_path.name)`; otherwise raise `FileValidationError` with the FFmpeg error details.
   - Return the probe dict (for optional use by callers).

4. Change the unsupported format `ValueError` to `UnsupportedFormatError`.

5. Change the `FileNotFoundError` to include a suggestion: "Check the file path and ensure the file exists."

Keep existing extension check BEFORE the FFmpeg probe (cheap check first, expensive second).

1. Add `tenacity>=9.0.0` to dependencies in `pyproject.toml` (add to the `dependencies` list).

2. Run `pip install tenacity>=9.0.0` to ensure it's available.

3. Create `transcribe/retry.py` with two retry decorator factories:

   a) `api_retry` — a pre-configured tenacity retry decorator for OpenAI API calls:
      - `retry=retry_if_exception_type((RateLimitError, APIConnectionError, APITimeoutError))` (import from `openai`)
      - `wait=wait_random_exponential(multiplier=1, min=4, max=60)`
      - `stop=stop_after_attempt(5)`
      - `reraise=True`
      - `before_sleep` callback that logs "Retrying API call (attempt {retry_state.attempt_number}/5)..." to stderr via `click.echo(..., err=True)` — this keeps user informed during retries even in quiet mode since retries are important.

   b) `ffmpeg_retry` — a retry decorator for FFmpeg/file I/O operations:
      - `retry=retry_if_exception_type((OSError, IOError))` — only OS-level I/O errors
      - `wait=wait_fixed(2)` — fixed 2s delay (I/O failures are usually permanent)
      - `stop=stop_after_attempt(3)`
      - `reraise=True`

   Export both as usable decorators (not factories — just decorated constants).

4. Update `transcribe/summarizer.py`:
   - Import `api_retry` from `transcribe.retry` and `APIRetryExhaustedError` from `transcribe.errors`.
   - Apply `@api_retry` decorator to `summarize_transcript()` function (the one that calls OpenAI directly).
   - In `summarize_long_transcript()`, wrap each `client.chat.completions.create()` call in a helper that has `@api_retry`. The simplest approach: extract the direct API call into a small inner function or use the decorator on summarize_transcript (which is already the single-call path). For the map-reduce loop calls, create a local `_api_call(client, messages, max_tokens)` helper decorated with `@api_retry`.
   - Update `check_api_key()` to raise `APIKeyMissingError` (from errors.py) instead of `RuntimeError`.

5. Update `transcribe/extractor.py`:
   - Import `ffmpeg_retry` from `transcribe.retry` and error classes from `transcribe.errors`.
   - Apply `@ffmpeg_retry` decorator to `extract_audio()`.
   - Change the `RuntimeError` raises to use appropriate error classes: `CorruptedFileError` for corrupt files, `NoAudioStreamError` for missing audio, `FileValidationError` for other FFmpeg errors.

IMPORTANT: Do NOT import `AuthenticationError` or similar non-transient errors in the retry filter. Only retry transient errors. The `reraise=True` ensures that after max attempts, the original exception propagates and the CLI can catch and display it.
